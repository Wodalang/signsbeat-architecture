💡
Architecture Summary — Claude（在 Anthropic 雲端）是 AI 大腦，負責理解問題與生成回答。你的主機只負責提供知識庫資料。Remote users 不直接連你的主機，而是透過 Claude 客戶端間接存取。
元件說明
元件說明
Remote Users (A/B/C)使用 Claude 客戶端（claude.ai、Claude Desktop、API 自建 App、或任何 MCP Client）的使用者
Claude AI (Anthropic Cloud)AI 大腦，託管在 Anthropic 雲端。負責接收用戶問題、決定呼叫哪個 MCP Tool、理解回傳內容、生成最終回答
MCP Server (server.py)跑在你的主機上的 Python 服務，透過 HTTPS + SSE 對外暴露 MCP 協議端點，供 Claude 呼叫
list_files()列出 Obsidian Vault 中所有 .md 檔案的路徑清單
read_file()讀取指定 .md 檔案的完整內容
search_content()根據關鍵字搜尋知識庫，回傳匹配的檔案片段
Obsidian Vault~/signsbeat/ 目錄下的 Markdown 知識庫，包含產品 wiki、技能文件等結構化內容

Key Insight — Claude 會自動判斷需要哪些資料。如果一次搜尋不夠，它可以連續呼叫多個 tool（例如先 search_content 再 read_file），直到收集足夠資訊才生成回答。
Flow 步驟詳解
步驟說明
Step 1: User Asks用戶在 Claude 客戶端輸入問題（例如 claude.ai 聊天介面）
Step 2: AI Decides ToolClaude 分析問題語意，自動決定要呼叫 search_content()、read_file() 或 list_files()，不需人工指定
Step 3: MCP CallClaude 透過 HTTPS/SSE 連線到你主機上的 MCP Server，發送 tool call 請求
Step 4: Return ResultsMCP Server 從 Obsidian Vault 讀取相關 .md 內容，回傳給 Claude
Step 5: Generate AnswerClaude 根據知識庫內容 + AI 理解，組合出最終回答送給用戶

⚠️
Bottleneck — 回覆速度的瓶頸不在你的主機，而在 Claude AI 的 token 生成速度（2-8 秒）。這跟一般 ChatGPT / Claude 聊天的體感速度一樣，用戶已經習慣這個等待時間，所以效能是可接受的。
🏠
Home Server Risks — 主要風險不是速度，而是穩定性：停電/斷網 = 服務中斷、家用上行頻寬有限（多人同時查詢可能變慢）、需要固定 IP 或 DDNS + SSL 憑證。建議初期用於內部團隊，驗證可行後再考慮遷到雲端。
效能對比
指標說明
MCP Server (<100ms)本地檔案讀取極快，即使知識庫有數百個 .md 檔也不是問題
Network (50-200ms)取決於你家的上行頻寬。中華電信光纖通常 50-100Mbps 上行，足夠應付文字傳輸
Claude AI (2-8s)最大瓶頸。Haiku 最快（~2s）、Sonnet 中等（~4s）、Opus 最慢但最聰明（~8s）。跟直接用 claude.ai 聊天一樣快
Concurrent UsersMCP Server 本身支援多人連線，但家用頻寬限制約 5-10 人同時使用。團隊內部使用綽綽有餘
Knowledge Base Size文字搜尋對 CPU 幾乎無負擔。即使 1000+ 個 .md 檔，搜尋也在 100ms 內完成
