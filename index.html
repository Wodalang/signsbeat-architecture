步驟名稱說明
(1)Router / Query Understanding解析使用者意圖與查詢語意，決定後續檢索策略
(2)Dense Retrieval包含 Date Search（時間範圍搜尋）與 Vector Search（向量語意搜尋），接收 User query data
(3)Sparse Retrieval與 Dense Retrieval 並行，透過關鍵字比對補充語意檢索的盲區
(4)Fusion融合 Dense Results 與 Sparse 結果，以排序演算法產出最相關的檢索資料
(5)Reasoning / Deep DiveLLM 基於融合後的檢索結果進行推理與深度分析
(6)Renderer / Final Answer將推理結果格式化為使用者可讀的最終回應
